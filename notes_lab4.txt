=== Comparaison JADX GUI vs JD-GUI ===

Différence 1 — Type de contenu affiché

JADX GUI :
Permet d’ouvrir directement l’APK complet. Il affiche le manifeste Android, les ressources, les fichiers XML, le fichier classes.dex et le code Java décompilé.

JD-GUI :
Permet d’ouvrir le fichier JAR généré avec dex2jar. Il affiche principalement les packages et classes Java, mais pas la structure Android complète de l’APK.

Différence 2 — Ressources Android

JADX GUI :
Affiche les ressources Android comme AndroidManifest.xml, strings.xml, activity_main.xml et menu_main.xml.

JD-GUI :
N’affiche pas directement les ressources Android de l’APK. Il affiche seulement le code Java contenu dans le JAR.

Différence 3 — Lisibilité du code Android

JADX GUI :
Le code Android est plus lisible, car JADX conserve souvent les références Android symboliques comme R.layout.activity_main ou R.id.edit_text.

JD-GUI :
Certaines références Android apparaissent sous forme d’identifiants numériques, par exemple :
setContentView(2130903040)
findViewById(2130837505)

Cela rend l’analyse moins intuitive.

Différence 4 — Noms des paramètres et reconstruction du code

JADX GUI :
Reconstruit un code plus adapté au contexte Android, avec une meilleure lisibilité générale.

JD-GUI :
Utilise des noms génériques comme paramString, paramBundle, paramView. Le code reste compréhensible, mais il est moins propre.

Différence 5 — Utilité dans le lab

JADX GUI :
Outil principal pour l’analyse statique Android, car il permet d’analyser l’APK complet.

JD-GUI :
Outil complémentaire utile pour confirmer le code Java après conversion DEX vers JAR.

Conclusion :
JADX GUI est plus adapté pour l’analyse statique complète d’un APK Android. JD-GUI est utile comme seconde approche pour vérifier les classes Java après conversion avec dex2jar.
